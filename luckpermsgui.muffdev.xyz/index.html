01Overview

LuckpermsGUI is a permission, group, and track editor that lives entirely inside chest inventories. It's built specifically for server admins and staff who'd rather click through a menu than remember exact /lp subcommand syntax — while still doing everything through LuckPerms' own public API, so nothing it does is invisible to LuckPerms itself.

Every change you make — toggling a permission, reordering a promotion ladder, setting a group's prefix — is written back through the same load-then-save path LuckPerms uses internally, and is logged to LuckPerms' own action log. Run /lp log afterward and changes made through this GUI look identical to changes made through LuckPerms' own commands.

02What It Does

Launcher

One command, /lpgui, opens a hub with six tiles: Users, Groups, Tracks, Contexts, Action Log, System Tools.

Users

  • Directory — browse or search every player LuckPerms knows about, online or not
  • Dashboard — groups, primary group, promote/demote along a track
  • Permission Nodes — toggle, remove, or add nodes with an optional expiry and a context restriction
  • Meta Keys — custom key/value data (ranks, placeholders, whatever your other plugins read)

Groups

  • Create, delete, and browse every group on the server
  • Weight, prefix, suffix, and display name, all editable in a couple of clicks
  • Parent groups and the exact same permission-node editor as Users

Tracks

  • Build promotion ladders visually — add groups in order, delete when done with one
  • Reorder by clicking the group you want to move, then clicking where it should go

Contexts

  • See the server's static context and any online player's currently active context set
  • Read-only — a diagnostic view, not an editor

Action Log

  • Browse the LuckPerms audit trail without leaving the GUI
  • Filter by actor or target name; force-refresh when you need the very latest entry

System Tools

  • Sync permissions across your network in one click
  • Jump straight into LuckPerms' own web editor
  • Purge locally cached permission data after a bulk change

03Requirements & Installation

You need: a Paper server (or a Paper-based fork) on Minecraft's Java 21 runtime, running Paper build 1.20.6 or newer, with LuckPerms already installed and working.

  1. Drop the plugin jar into your server's plugins/ folder.
  2. Make sure LuckPerms is in the same folder — LuckpermsGUI won't start without it, and will tell you clearly in the console if it can't find it.
  3. Restart the server (a plugin reload isn't enough for a fresh install).
  4. Grant luckpermsgui.admin to whichever staff should have access — it defaults to operators only.
  5. Run /lpgui in-game to open the Launcher.

Chest inventories can't take multi-field forms, so anywhere the GUI needs more than one value, it takes a single compact line typed in chat. Type cancel, or click the Cancel link, to back out — every prompt also times out after 30 seconds on its own.

06How It Works

Every permission node is one of exactly three states — nothing invented, this is just how LuckPerms itself stores permissions:

True  granted  ·  Negated  explicitly denied  ·  Unset  no node at all

Left-click a permission to flip True/Negated; right-click to unset it. The same rule holds everywhere in the GUI: right-click generally means remove, left-click generally means toggle or select. Every paginated list uses the same bottom row too, so once you've used one screen you've used them all — page controls in the middle, and the very last slot is always either Back (if you navigated in from somewhere) or Exit (if this is where your session started).

07Limitations, Honestly

  • Paper 1.20.6+ only — the plugin uses Paper's modern command registration API, which doesn't exist on 1.20.4 or earlier.
  • One context restriction per permission node, not an arbitrary set.
  • Prefix and suffix priority is fixed when set through the GUI — setting a new one replaces any existing one rather than stacking multiple priorities.
  • Group deletion doesn't hard-block on "this is someone's primary group" — that would require scanning every stored player. The dashboard shows track membership before you delete, and deletion always needs two clicks, but there's no automatic prevention.
  • Meta values can't carry an expiry through the simple chat syntax — a value can contain almost anything, including something that looks like a duration, so the plugin doesn't try to guess.
  • The action log has no server-side search — LuckPerms' own API returns the whole log at once, so filtering happens locally against a copy that refreshes at most every 45 seconds.

08Built With Care

Every LuckPerms API call this plugin makes was checked against the real, compiled API — not assumed from memory — before being written into the plugin. Two real interaction bugs were caught and fixed before release by tracing through concrete examples rather than trusting the first version that compiled: an off-by-one in the track-reordering math, and a rare case where a late chat response could reopen a stale menu on top of whatever a player had already navigated to. Every LuckPerms call runs off the main server thread, with results always hopped back before touching an inventory or sending a message, so nothing here can freeze your server waiting on permission data.
